Updating Documentation for Achilles SoC SOM

This guide explains how to update documentation in the reflexces/meta-achilles repository and deploy changes to GitHub Pages.

Prerequisites

  • Python 3.6+ installed (python3 --version)
  • Git installed and configured with GitHub access (git --version)
  • Push permissions to the reflexces/meta-achilles repository

Steps to Update Documentation

1. Clone the Repository and Checkout docs Branch

git clone https://github.com/reflexces/meta-achilles.git
cd meta-achilles
git checkout docs

2. Install MkDocs and Required Extensions

Try global install first:

pip3 install mkdocs pymdown-extensions

If you get the error "externally-managed-environment", use one of these options:

Option A - Virtual Environment (Recommended): 

python3 -m venv mkdocs-venv
source mkdocs-venv/bin/activate
pip install mkdocs pymdown-extensions

Option B - Force global install: 

pip3 install --break-system-packages mkdocs pymdown-extensions

Verify installation:

mkdocs --version

3. Edit Documentation

Edit the relevant .md file:

nano docs/resources.md

Common files: - index.md - start-here.md - hardware.md - software.md - program-emmc.md - resources.md

Tips: - Use 4 spaces for sub-bullets (important for proper rendering) - Use correct image paths, e.g. images/achilles_design_flow_gsrd_2022.06.png - Maintain header consistency (# for titles, ## for sections)

4. Test Changes Locally

mkdocs build --clean
mkdocs serve

Open http://127.0.0.1:8000 in your browser and verify the changes.

5. Commit Changes

git add docs/your-edited-file.md
git commit -m "Fix formatting on XYZ page"
git push origin docs

6. Deploy to GitHub Pages

mkdocs gh-deploy --force

This updates the live site at: https://reflexces.github.io/meta-achilles/

7. Verify Live Site

Visit https://reflexces.github.io/meta-achilles/ and confirm your changes appear correctly.

Notes

  • Only commit source files (.md, .yml, .css, images). Do not commit the site/ folder.
  • Image filenames should not contain spaces.
  • For internal-only notes, use update-docs-INTERNAL.md.

Questions?
